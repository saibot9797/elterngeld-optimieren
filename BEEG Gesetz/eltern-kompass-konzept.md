# Eltern-Kompass

## Der Elterngeld-Optimierer – "Taxfix für werdende Eltern"

---

## Executive Summary

**Eltern-Kompass** ist der erste Elterngeld-Optimierer Deutschlands. Wie Taxfix bei der Steuererklärung holen wir das Maximum an Familienleistungen raus – mit einem klaren Versprechen: **Durchschnittlich 2.400€ mehr Elterngeld** durch intelligente Aufteilung und rechtzeitige Anträge.

### Das Problem

Werdende Eltern verschenken tausende Euro durch:
- **Falsche Elterngeld-Aufteilung** (Basiselterngeld vs. ElterngeldPlus vs. Partnerschaftsbonus)
- **Suboptimale Steuerklassen-Wahl** vor der Elternzeit
- **Verpasste Fristen** (Elterngeld nur 3 Monate rückwirkend!)
- **Unbekannte Zusatzleistungen** (Landeserziehungsgeld Bayern, Kinderzuschlag, Wohngeld)
- **Fehlerhafte Anträge** die zu Rückforderungen führen

**Marktvalidierung:**
- Professionelle Elterngeld-Beratung kostet **100-300€** und ist oft ausgebucht
- Stiftung Warentest bestätigt: Beratung bringt **tausende Euro mehr**
- Ein dokumentierter Fall: **8.056€ mehr Elterngeld** durch Optimierung

**Eltern unter Druck (Forsa-Studie 2025):**
- **51%** der Eltern fühlen sich durch Pflichten im Alltag belastet
- **33%** der Mütter und **24%** der Väter stufen sich als "stark belastet" ein
- Nur **20%** fühlen sich wenig oder gar nicht belastet
- Quelle: Körber-Stiftung, Forsa-Umfrage 2025

### Die Lösung

Eine App, die wie ein persönlicher Elterngeld-Berater funktioniert:
1. Analysiert die individuelle Situation in 10 Minuten
2. Berechnet alle möglichen Aufteilungs-Szenarien
3. Zeigt den **optimalen Weg** mit konkretem €-Betrag
4. Führt durch alle Anträge mit vorausgefüllten Formularen
5. Erinnert automatisch an Fristen

**Value Proposition:**
> "Kostenlos testen. Nur 79€ zahlen, wenn wir dir mindestens 500€ mehr rausholen."

  if (pregnancy.is_married) {
    const currentTax = calculateElterngeld(pregnancy)
    const optimizedTax = calculateElterngeldWithTaxSwitch(pregnancy)

    if (optimizedTax - currentTax > 100) {
      optimizations.push({
        type: 'tax_class_switch',
        potential_gain: optimizedTax - currentTax,
        description: 'Steuerklassenwechsel zu 3/5',
        how_to: 'Wechsel mindestens 7 Monate vor Mutterschutz...',
        deadline: subMonths(pregnancy.due_date, 9)
      })
    }
  }

  // 2. ElterngeldPlus vs. Basis
  const basisScenario = calculateBasisOnly(pregnancy)
  const plusScenario = calculateWithPlus(pregnancy)
  const mixedScenario = calculateOptimalMix(pregnancy)

  const bestScenario = Math.max(basisScenario, plusScenario, mixedScenario)
  // ... weitere Logik

## ⚠️ Kritische Annahmen & potenzielle Denkfehler

> **Wichtig:** Die folgenden Punkte müssen vor Launch geklärt/korrigiert werden.

### 1. Steuerklassenwechsel-Timing (Zeile 173)

**Problem:** "Steuerklassenwechsel 3 Monate vor Mutterschutz" ist sehr wahrscheinlich **zu spät** bzw. so pauschal **falsch**.

**Realität:**
- Relevant ist, welche Steuerklasse in der **Mehrzahl der Bemessungsmonate** galt
- Faustregel: Eher **~7 von 12 Monaten** vor Geburt müssen in der günstigeren Steuerklasse liegen
- Unterschiede je nach Konstellation:
  - Mutter vs. Vater (wer optimiert?)
  - Angestellt vs. Selbstständig (unterschiedliche Bemessungszeiträume)

**To-Do:** Differenzierte Logik im Algorithmus implementieren, nicht pauschale "3 Monate"-Aussage.

### 2. ElterngeldPlus ≠ automatisch "mehr Geld" (Zeile 174)

**Problem:** "ElterngeldPlus statt Basiselterngeld" als direkte €-Optimierung dargestellt.

**Realität:**
- **Ohne paralleles Teilzeit-Einkommen** ist ElterngeldPlus meist nur eine **Streckung/Planungsoption**, nicht automatisch "mehr Geld"
- "Mehr Geld" entsteht typischerweise erst durch:
  - Zuverdienst während ElterngeldPlus-Bezug
  - Partnerschaftsbonus-Regeln (beide 24-32h arbeiten)

**To-Do:** Optimierer muss Teilzeit-Szenarien abfragen; ElterngeldPlus nur als Vorteil zeigen, wenn tatsächlich Zuverdienst geplant.

### 3. Unbelegte Marketing-Claims (Zeile 9, 53)

**Problem:** "Durchschnittlich 2.400€ mehr" und "erster Elterngeld-Optimierer Deutschlands" sind harte Claims ohne Nachweis.

**Risiken:**
- **Rechtlich:** Abmahnung wegen irreführender Werbung (§5 UWG)
- **Vertrieblich:** Vertrauensverlust, wenn Claims nicht haltbar
- **Inhaltlich fragil:** Der "Durchschnitt" kann durch viele 0€-Fälle verzerrt sein

**To-Do:**
- Entweder belastbare Daten sammeln (z.B. aus Beta-Phase) ODER
- Formulierung abschwächen: "Bis zu X€ mehr" / "Viele Eltern verschenken X€"
- "Erster" nur behaupten, wenn nachweisbar – ansonsten streichen

### 4. Definition der 500€-Garantie (Zeile 41)

**Problem:** "Nur zahlen, wenn mind. 500€ mehr" braucht eine wasserdichte Definition der Vergleichsbasis.

**Offene Fragen:**
- Was ist "Standard-Aufteilung"? (12+2 Basiselterngeld? 50/50?)
- Wer definiert das "Vorher"? Der User? Wir?
- **Kernproblem:** Die Elterngeldstelle entscheidet final – wir können nur "berechnetes Potenzial" garantieren, nicht den tatsächlichen Bescheid

**To-Do:**
- Klare Definition in AGB: "Vergleich zu X-Aufteilung"
- Garantie auf "berechnetes Optimierungspotenzial" beziehen, nicht auf Bescheid
- Juristisch prüfen lassen

### 5. Formular-Komplexität unterschätzt (Zeile 443)

**Problem:** "Bundeseinheitliches Formular → nur 6 Formular-Sets" klingt zu einfach.

**Realität:** Selbst bei Standardformularen gibt es viele Varianten/Anlagen je nach Fall:
- Selbstständige (andere Nachweise, Einnahmen-Überschuss-Rechnung)
- Bonuszahlungen / Einmalzahlungen (Ausklammerungstatbestände)
- Teilzeit während Elterngeld (Zuverdienst-Berechnung)
- PKV vs. GKV (unterschiedliche Nachweise)
- Mischeinkünfte, Minijobs, etc.

**To-Do:** Formular-Engine muss situationsabhängige Anlagen/Nachweise berücksichtigen – nicht nur 6 Basis-Templates.

### 6. Legal/Compliance: Grenze zur Steuer-/Rechtsberatung

**Problem:** Strategische Empfehlungen wie "Steuerklasse wechseln" oder "optimale Aufteilung" + konkrete Handlungsanweisungen können **zu nah an erlaubnispflichtige Steuer-/Rechtsberatung** sein.

**Risiko:** Ein Disclaimer allein reicht möglicherweise nicht, wenn das Produkt de facto berät.

**Optionen:**
1. **Reines Rechen-/Szenario-Tool:** Zeigt nur Zahlen, keine Empfehlungen → weniger wertvoll, aber sicher
2. **Beratung über Partner:** Kooperation mit Steuerberatern/Rechtsanwälten für "echte" Empfehlungen
3. **Produktisierung:** Rechtlich früh klären, was erlaubt ist (z.B. "Informationen" vs. "individuelle Beratung")

**To-Do:** Vor Launch juristisch prüfen lassen, wie weit wir gehen dürfen. Ggf. Formulierungen anpassen ("Szenario A bringt X€" statt "Du solltest X tun").

### Wer muss welche Daten angeben?

#### Grundprinzip der Elterngeld-Berechnung

**Wichtig:** Elterngeld wird **pro Person individuell** berechnet. Jeder Elternteil, der Elterngeld beziehen möchte, benötigt eine eigene Berechnung basierend auf **seinem eigenen** Einkommen vor der Geburt.

| Datenkategorie | Mutter | Partner | Begründung |
|----------------|--------|---------|------------|
| **Einkommen (Brutto/Netto)** | Eigenes | Eigenes | Elterngeld = % vom **eigenen** Einkommensverlust |
| **Steuerklasse** | Eigene | Eigene | Beeinflusst **eigenes** Elterngeld-Netto |
| **Beschäftigungsart** | Eigene | Eigene | Bestimmt **eigenen** Bemessungszeitraum |
| **Geplante Elternzeit** | Eigene | Eigene | Für Aufteilungs-Optimierung |
| **Zu versteuerndes Einkommen** | **GEMEINSAM** | **GEMEINSAM** | Einkommensgrenze 175.000€ gilt für **beide zusammen** |

### 2. EINKOMMENSDATEN FÜR BERECHNUNG

#### 2.1 Wessen Einkommen wird für wessen Elterngeld verwendet?

```
┌─────────────────────────────────────────────────────────────────┐
│  WICHTIGES PRINZIP                                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Elterngeld MUTTER = 65-67% vom Einkommen der MUTTER           │
│  Elterngeld PARTNER = 65-67% vom Einkommen des PARTNERS        │
│                                                                 │
│  → Jede Person bekommt Elterngeld basierend auf dem            │
│    EIGENEN Einkommensverlust, NICHT vom Haushaltseinkommen!    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 2.2 Bemessungszeitraum (§2b BEEG)

| Person | Bemessungszeitraum | Besonderheit |
|--------|-------------------|--------------|
| **Mutter (angestellt)** | 12 Monate vor **Mutterschutzbeginn** | Mutterschutz beginnt 6 Wochen vor ET |
| **Partner (angestellt)** | 12 Monate vor **Geburt** | Kein Mutterschutz → direkt vor Geburt |
| **Selbstständige** | Letzter abgeschlossener Steuerbescheid | Kalenderjahr, nicht Monate |

**Beispiel bei ET 01.08.2026:**

### 3. STEUERKLASSEN-REGELUNG (§2c, §2e BEEG)

#### 3.1 Warum die Steuerklasse wichtig ist

Die Steuerklasse beeinflusst das **"Elterngeld-Netto"**, das die Elterngeldstelle berechnet. Höhere Steuerklasse (III) = höheres Netto = höheres Elterngeld.

#### 3.2 Welche Steuerklasse für wen?

| Frage | Antwort |
|-------|---------|
| Wessen Steuerklasse beeinflusst wessen Elterngeld? | **Nur die eigene!** |
| Mutter Steuerklasse III → | Höheres Elterngeld für **Mutter** |
| Partner Steuerklasse III → | Höheres Elterngeld für **Partner** |
| Beide Steuerklasse IV → | Neutraler Einfluss für beide |

#### 3.3 Überwiegende Steuerklasse (§2c Abs. 3 BEEG)

**Kritische Regel:** Nicht der letzte Monat zählt, sondern die **Mehrheit der Monate**!

| Situation | Verwendete Steuerklasse |
|-----------|------------------------|
| Steuerklasse A in 7+ Monaten | Steuerklasse A |
| Steuerklasse A in 6 Monaten, B in 6 Monaten | Steuerklasse im **letzten Monat** |
| Steuerklasse VI | Wird **NIE** berücksichtigt |

**Optimierungs-Timing:**
```
Für maximales Elterngeld der MUTTER:
→ Mutter muss mind. 7 Monate VOR Mutterschutzbeginn Steuerklasse III haben

Für maximales Elterngeld des PARTNERS:
→ Partner muss mind. 7 Monate VOR Geburt Steuerklasse III haben

ACHTUNG: Nur EINER kann Steuerklasse III haben!
→ Optimierung für die Person mit dem höheren Einkommen ODER
→ Optimierung für die Person, die mehr Monate Elterngeld bezieht
```

#### 3.4 Benötigte Steuerklassen-Daten

| Datenpunkt | Mutter | Partner | Warum |
|------------|--------|---------|-------|
| **Aktuelle Steuerklasse** | ✓ | ✓ | Basis für Berechnung |
| **Steuerklasse je Monat im Bemessungszeitraum** | ✓ | ✓ | Für "überwiegende" Ermittlung |
| **Datum letzter Steuerklassenwechsel** | ✓ | ✓ | Prüfung ob Optimierung noch möglich |
| **Kirchensteuerpflicht** | ✓ | ✓ | Zusätzlicher Abzug |

### 11. BERECHNUNGSFORMELN (BEEG-konform)

#### 11.1 Elterngeld-Netto Berechnung

```typescript
// Vereinfachte Formel nach §2c, §2e, §2f BEEG

function berechneElterngeldNetto(
  brutto: number,
  steuerklasse: 1 | 2 | 3 | 4 | 5,
  kirchensteuer: boolean,
  gkvPflichtig: boolean,
  rvPflichtig: boolean,
  alvPflichtig: boolean
): number {
  // 1. Werbungskosten-Pauschale abziehen (§2c)
  const werbungskosten = 1230 / 12 // = 102,50€/Monat
  const bereinigtesBrutto = brutto - werbungskosten

  // 2. Steuerabzug nach Steuerklasse (§2e)
  const steuerabzug = berechneSteuer(bereinigtesBrutto, steuerklasse, kirchensteuer)

  // 3. Sozialabgaben-Pauschalen (§2f)
  let sozialabgaben = 0
  if (gkvPflichtig) sozialabgaben += bereinigtesBrutto * 0.09 // 9% KV+PV
  if (rvPflichtig) sozialabgaben += bereinigtesBrutto * 0.10  // 10% RV
  if (alvPflichtig) sozialabgaben += bereinigtesBrutto * 0.02 // 2% ALV

  // 4. Elterngeld-Netto
  return bereinigtesBrutto - steuerabzug - sozialabgaben
}
```

#### 11.2 Ersatzrate nach Einkommenshöhe

```typescript
// Ersatzrate nach §2 Abs. 2 BEEG

function berechneErsatzrate(nettoVorGeburt: number): number {
  if (nettoVorGeburt > 1240) {
    // Absenkung: -0,1% je 2€ über 1240€, bis min. 65%
    const ueber1240 = nettoVorGeburt - 1240
    const absenkung = Math.floor(ueber1240 / 2) * 0.001
    return Math.max(0.65, 0.67 - absenkung)
  } else if (nettoVorGeburt >= 1000) {
    // Zwischen 1000€ und 1240€: gestaffelt 65%-67%
    const unter1240 = 1240 - nettoVorGeburt
    const erhoehung = Math.floor(unter1240 / 2) * 0.001
    return Math.min(0.67, 0.65 + erhoehung)
  } else {
    // Unter 1000€: Erhöhung bis 100%
    const unter1000 = 1000 - nettoVorGeburt
    const erhoehung = Math.floor(unter1000 / 2) * 0.001
    return Math.min(1.0, 0.67 + erhoehung)
  }
}
```

#### 11.3 Elterngeld Berechnung

```typescript
// Hauptberechnung nach §2 BEEG

function berechneElterngeld(
  nettoVorGeburt: number,
  nettoWaehrendElternzeit: number = 0
): number {
  const einkommensverlust = nettoVorGeburt - nettoWaehrendElternzeit
  const ersatzrate = berechneErsatzrate(nettoVorGeburt)

  let elterngeld = einkommensverlust * ersatzrate

  // Grenzen anwenden
  const MINIMUM = 300
  const MAXIMUM = 1800

  return Math.max(MINIMUM, Math.min(MAXIMUM, elterngeld))
}
```
